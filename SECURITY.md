# Security Policy

## Reporting Security Issues

Report security issues privately to the maintainers before opening a public issue. If no private contact is listed, open a minimal public issue asking for a security contact without sharing exploit details.

## Do Not Share Secrets

Never share API keys, tokens, passwords, private keys, session cookies, customer data, private screenshots or private repository content in public issues, pull requests, screenshots or logs.

## Accidentally Committed Secrets

If a secret is accidentally committed to Git, rotate or revoke it immediately. Removing the value from the latest commit is not enough because it may remain in Git history, forks, caches or local clones.
